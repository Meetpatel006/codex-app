E2E CRYPTO HANDSHAKE FLOW (new — from remodex relay source)
01
Bridge generates Ed25519 identity keypair once, persists to ~/.myrelay/identity.json. This is the stable bridge identity.
02
On bridge up: generate random sessionId + X25519 ephemeral keypair for this session. QR encodes relay_url | sessionId | ed25519_pubkey | expiry.
03
Bridge connects to relay as x-role: mac. Relay creates in-memory session room, stores macDeviceId → sessionId in trusted index.
04
iPhone scans QR, connects to /relay/{sessionId} as x-role: iphone. If Mac session not live → relay closes with 4002. If live → relay binds iPhone to session.
05
Signed handshake via secure-control messages (plaintext at relay layer — relay can see these). Both sides exchange X25519 ephemeral pubkeys, each signs with their Ed25519 identity key.
06
Both sides run X25519(my_ephemeral_private, peer_ephemeral_public) → shared secret. Feed into HKDF-SHA256 → derive AES-256-GCM keys. Relay is now blind to all app payloads.
07
Trusted reconnect: on later app reopen, iPhone calls POST /v1/trusted/session/resolve with a signed request (Ed25519 sig + nonce + timestamp). Relay verifies freshness, returns current live sessionId. No QR re-scan.
